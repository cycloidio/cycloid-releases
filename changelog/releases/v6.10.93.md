# [v6.10.93] _2026-06-11_

## Console
### **ADDED**
- **Authentication**: OIDC provider settings now support an optional `Discovery URL` override and an `Allow plain HTTP` toggle, enabling split-network deployments where the Cycloid backend reaches the IdP via a different network path than user browsers (private DMZ, K8s cluster DNS, split-horizon DNS) [FE-1468](https://linear.app/cycloid/issue/FE-1468)
- **Authentication**: OIDC group mapping — organizations can map identity-provider groups to Cycloid teams, with an admin UI to create and edit mappings and configure per-org OIDC settings (default role, OIDC-managed mode, no-match policy) [FE-1435](https://linear.app/cycloid/issue/FE-1435)
- **Authentication**: OIDC logins that resolve to no organization access now land on a dedicated no-access page instead of showing a raw SSO error on the login form [FE-1448](https://linear.app/cycloid/issue/FE-1448)
### **FIXED**
- **Authentication**: Fix role editor overwriting resources on unscoped rules when editing unrelated permissions [ENGFE-482](https://linear.app/cycloid/issue/ENGFE-482)
- **Authentication**: OIDC group mappings are now visible for organisations where the OIDC integration is in a disabled state but OIDC management is active (oidc_managed: true) [FE-1435](https://linear.app/cycloid/issue/FE-1435)
### **SECURITY**
- **Authentication**: OIDC account claiming no longer sends a client-controlled email_verified flag; the backend now derives it server-side from the verified OIDC token [BE-1478](https://linear.app/cycloid/issue/BE-1478)
### **FIXED**
- **Credentials**: Generated canonicals are no longer truncated to 50 characters, so longer credential names keep their full canonical. [ENGFE-492](https://linear.app/cycloid/issue/ENGFE-492)
### **CHANGED**
- **Environments**: Reword the project-level environment delete action as "Remove from project", since it unlinks the env rather than deleting it. [ENGFE-485](https://linear.app/cycloid/issue/ENGFE-485)
### **FIXED**
- **Environments**: Fix environment rename failing, and stop showing a success message when the update fails. [ENGFE-484](https://linear.app/cycloid/issue/ENGFE-484)
- **Environments**: Populate the project-page environment form once its data loads. [ENGFE-486](https://linear.app/cycloid/issue/ENGFE-486)
- **Organizations**: Batch permission checks and cache evaluated actions to reduce can_do request fan-out and page load latency [ENG-156](https://linear.app/cycloid/issue/ENG-156)
### **ADDED**
- **Project**: Link to organization level environment settings from project settings [FE-1495](https://linear.app/cycloid/issue/FE-1495)
- **Roles**: Hide the Projects, Stacks, Resources, Observability, FinOps and Security menu sections from users whose roles don't grant the matching menu permission [FE-1491](https://linear.app/cycloid/issue/FE-1491)
### **FIXED**
- **Roles**: Users with read-only roles are no longer redirected to an "Access denied" page when opening the dashboard or other pages. Sections they can't access simply stay hidden instead of blocking the whole page. [ENGFE-491](https://linear.app/cycloid/issue/ENGFE-491)
- **Roles**: Buttons and actions you don't have permission to use are now hidden instead of leading to an "access denied" or "not found" page when clicked. [ENGFE-490](https://linear.app/cycloid/issue/ENGFE-490)
### **ADDED**
- **Service Catalog**: Org settings — new "Stack Versioning" section allowing admins to configure include/exclude glob patterns for branch visibility [PROD-740](https://linear.app/cycloid/issue/PROD-740)
- **Service Catalog**: Warn when a catalog repository branch will be hidden from end-users by the org stack version patterns [PROD-740](https://linear.app/cycloid/issue/PROD-740)
- **StackForms**: Add dynamic_search support to skip upfront values_ref fetch and load options on-demand as the user types. [FE-1474](https://linear.app/cycloid/issue/FE-1474)
- **Plugins**: Add clickable timestamps in plugin logs to share specific lines [FE-1443](https://linear.app/cycloid/issue/FE-1443)
### **CHANGED**
- **Plugins**: Show plugin manager connection status as a colored dot indicator instead of a tag [FE-1490](https://linear.app/cycloid/issue/FE-1490)
### **FIXED**
- **Plugins**: show the widget-unavailable fallback instead of a raw error when an iframe widget plugin is not running [FE-1481](https://linear.app/cycloid/issue/FE-1481)
- **Plugins**: Fix plugin update sending installed version id instead of latest, causing a silent no-op redeploy [ENG-159](https://linear.app/cycloid/issue/ENG-159)
- **Plugins**: Build the component plugin relations form from each widget's declared relations [FE-1497](https://linear.app/cycloid/issue/FE-1497)
## API
### **ADDED**
- **Authentication**: Added menu permissions for the remaining sidebar sections [BE-1502](https://linear.app/cycloid/issue/BE-1502)
### **FIXED**
- **Authentication**: Add menu permissions to existing custom roles [BE-1507](https://linear.app/cycloid/issue/BE-1507)
- **Components**: Component list is no longer empty for users with a project-scoped custom role (e.g. maintainer): the list and count queries filtered the project entitlement against the component canonical instead of the project canonical, so scoped roles never matched and saw zero components. [ENGBE-271](https://linear.app/cycloid/issue/ENGBE-271)
- **Environments**: Fix 403 on environment create/update/delete for custom roles after v6.10.29. The env governance release moved env mutations to new standalone endpoints guarded by organization:environment:{create,update,delete} but only seeded these actions onto built-in roles; custom roles holding the legacy organization:project:update were left with no matching grant. Env mutation endpoints now accept either the new env-specific action or the legacy organization:project:update as a backwards-compatibility fallback. [ENGBE-276](https://linear.app/cycloid/issue/ENGBE-276)
### **CHANGED**
- **Members**: An organization is now NOT automatically removed when its last member gets deleted. [BE-1471](https://linear.app/cycloid/issue/BE-1471)
### **ADDED**
- **Organizations**: Add batch can_do_batch endpoint to check multiple authorization actions in a single request
### **FIXED**
- **Roles**: Updating a role no longer fails with "Has to define a new role containing members with admin rights on the organization" in organizations that grant admin exclusively through teams. The anti-lockout guard now only runs when the role being updated currently grants OrganizationAdmin, mirroring the existing delete behavior. [ENGBE-272](https://linear.app/cycloid/issue/ENGBE-272)
- **Roles**: A user with permissions scoped to a resource is now not able to create their own resources [ENG-160](https://linear.app/cycloid/issue/ENG-160)
- **Plugin**: Include plugin relation data in component plugin widgets response [BE-000](https://linear.app/cycloid/issue/BE-000)
